// how node.js actually runs your code

Application → V8 → Bindings → libuv owns the Event Loop

Horizontal flow, left to right — just like real Node.js system diagrams. libuv is the single C library that encapsulates the Event Queue, Event Loop, Thread Pool, and OS Async I/O.

V8 Engine
Call stack, heap, microtask queue — where your JS actually runs.
Node.js APIs
JS-land wrappers: fs, http, timers, crypto...
Bindings
C++ glue — decides if a call goes sync or async.
Sync libuv Call
Same libuv function as the async version, run inline on the calling thread — skips the thread pool & event loop, not libuv itself.
Event Queue
Callbacks waiting for their turn on the Call Stack.
Event Loop
The engine libuv runs — cycles through 6 phases forever.
Thread Pool
4 default threads for blocking work: fs, crypto, zlib, dns.
OS Async I/O
epoll/kqueue/IOCP — sockets, zero extra threads.
Worker Threads
Separate isolate + own event loop, for parallel JS — not the same as the Thread Pool.
